# 分析报告功能使用说明

## 功能概述

新增了历史分析报告功能，可以保存和查看所有生成的AI分析报告。每个用户只能看到自己生成的报告历史。

## 实现内容

### 1. 后端改动

#### 数据库表模型
在 `backend/app.py` 中的 `AnalysisReport` 模型：
- `id`: 主键，自增
- `username`: 生成报告的用户名（新增）
- `dealer_code`: 经销商代码（来自销量驱动页面的经销商选择器）
- `report_date`: 报告生成日期
- `selected_cards`: 选中的卡片数据（JSON格式）
- `report_content`: 报告内容（Markdown格式）
- ~~`created_at`~~: 已删除此字段

#### API接口
- `POST /api/analysis-reports` - 保存分析报告（需要 username 和 dealer_code）
- `GET /api/analysis-reports?username=xxx` - 获取分析报告列表（按用户名筛选）
- `GET /api/analysis-reports/<id>` - 获取单个报告详情
- `DELETE /api/analysis-reports/<id>` - 删除报告

### 2. 前端改动

#### 修改 DealerDashboard.vue
- 在 `ReportModal` 组件中传递 `dealerCode` 属性
- `dealerCode` 来自页面中的经销商选择器（`selectedCode`）

#### 修改 ReportModal.vue
- 添加 `dealerCode` 属性接收
- 在报告生成成功后自动保存到数据库
- 保存时同时记录：
  - `username`: 当前登录用户的用户名
  - `dealer_code`: 当前选择的经销商代码

#### 修改 AnalysisReports.vue 页面
- 展示当前用户生成的所有历史分析报告
- 普通用户只能看到自己的报告
- 管理员可以看到所有用户的报告
- 显示用户名和经销商代码信息
- 支持查看报告详情、删除、下载和复制

#### 更新路由配置
在 `router/index.js` 中添加了新路由：
```javascript
{
  path: 'analysis-reports',
  name: 'analysisReports',
  component: AnalysisReports
}
```

#### 更新侧边栏菜单
在 `SideMenu.vue` 中的"仪表盘"菜单下添加了"分析报告"选项

## 使用步骤

### 1. 初始化/更新数据库表

在后端目录运行初始化脚本（会删除旧表并创建新表）：

```bash
cd backend
python init_analysis_report_table.py
```

**注意**：运行此脚本会删除旧的 `analysis_report` 表及其所有数据，请谨慎操作！

### 2. 启动后端服务

```bash
cd backend
python app.py
```

### 3. 启动前端服务

```bash
cd my_vue
npm run serve
```

### 4. 使用功能

1. 登录系统（例如：用户 user1 登录）
2. 在仪表盘的"经销商选择"中选择要分析的经销商（例如：DS001）
3. 选择卡片并生成AI分析报告
4. 报告生成后会自动保存到数据库，记录：
   - username: user1（当前登录用户）
   - dealer_code: DS001（当前选择的经销商）
5. 点击侧边栏"仪表盘" -> "分析报告"查看自己的历史报告
6. 可以查看详情、下载、复制或删除报告

## 重要说明

### 数据记录方式
每条报告记录包含：
- **username**: 生成报告的用户（登录用户）
- **dealer_code**: 被分析的经销商（从经销商选择器选择）

### 权限说明
- **普通用户**: 只能查看自己生成的分析报告（按 username 筛选）
- **管理员用户**: 可以查看所有用户生成的分析报告

### 使用场景示例
1. 用户 A 登录，选择经销商 DS001，生成报告
   - 记录：username=A, dealer_code=DS001
2. 用户 A 登录，选择经销商 DS002，生成报告
   - 记录：username=A, dealer_code=DS002
3. 用户 B 登录，选择经销商 DS001，生成报告
   - 记录：username=B, dealer_code=DS001

结果：
- 用户 A 可以看到自己的 2 条报告（DS001 和 DS002）
- 用户 B 可以看到自己的 1 条报告（DS001）
- 管理员可以看到所有 3 条报告

## 注意事项

1. 确保 MySQL 数据库正在运行
2. 确保数据库连接配置正确（`app.py` 中的数据库URI）
3. 运行初始化脚本会删除旧表的所有数据
4. 报告内容以 Markdown 格式存储，前端使用 `marked` 库渲染
5. 选中的卡片数据以 JSON 字符串格式存储

## 技术栈

- **后端**: Flask + SQLAlchemy + MySQL
- **前端**: Vue.js + Vue Router + marked.js
- **样式**: 自定义 CSS（渐变背景、卡片设计）
