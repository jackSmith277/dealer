# 环境变量配置说明

## 📋 概述

现在系统已经支持使用环境变量来管理 DeepSeek API Key，这样更安全、更灵活。

## ✅ 已完成的配置

### 1. 文件结构

```
my_vue/
├── .env                    # 实际配置文件（已创建，包含你的API Key）
├── .env.example           # 配置示例文件（供参考）
├── .gitignore             # 已添加 .env 到忽略列表
└── src/
    └── DS/
        └── deepseek.js    # 已更新为支持环境变量
```

### 2. 代码更新

`deepseek.js` 现在会：
1. **优先读取环境变量** `VITE_DEEPSEEK_API_KEY`
2. **如果没有环境变量**，则使用硬编码的值（开发环境备用）

```javascript
// 优先使用环境变量，如果没有则使用硬编码的值
const DEEPSEEK_API_KEY = import.meta.env?.VITE_DEEPSEEK_API_KEY || 'sk-829e1bc28fe145beae4780d96fd5d2df';
```

### 3. .env 文件内容

```env
VITE_DEEPSEEK_API_KEY=sk-829e1bc28fe145beae4780d96fd5d2df
VITE_DEEPSEEK_API_URL=https://api.deepseek.com/v1/chat/completions
VITE_DEEPSEEK_MODEL=deepseek-chat
VITE_DEEPSEEK_TEMPERATURE=0.7
VITE_DEEPSEEK_MAX_TOKENS=4000
```

### 4. .gitignore 更新

已添加 `.env` 到忽略列表，防止 API Key 被提交到 Git 仓库。

## 🎯 使用方式

### 方式一：使用环境变量（推荐）

1. **.env 文件已创建**，包含你的 API Key
2. **重启开发服务器**（如果正在运行）：
   ```bash
   # 停止当前服务器（Ctrl+C）
   # 重新启动
   npm run serve
   ```
3. **系统会自动读取** .env 文件中的配置

### 方式二：直接使用硬编码（当前可用）

如果 .env 文件不存在或读取失败，系统会自动使用硬编码的 API Key，所以**不影响当前使用**。

## 🔒 安全性提升

### 之前（不安全）
```javascript
// API Key 直接写在代码中
const DEEPSEEK_API_KEY = 'sk-829e1bc28fe145beae4780d96fd5d2df';
```

**问题**：
- ❌ API Key 暴露在代码中
- ❌ 提交到 Git 后所有人都能看到
- ❌ 难以管理多个环境的配置

### 现在（安全）
```javascript
// 从环境变量读取
const DEEPSEEK_API_KEY = import.meta.env?.VITE_DEEPSEEK_API_KEY || '备用值';
```

**优势**：
- ✅ API Key 存储在 .env 文件中
- ✅ .env 文件不会被提交到 Git
- ✅ 每个开发者可以有自己的配置
- ✅ 生产环境可以使用不同的 API Key

## 🔧 配置项说明

### VITE_DEEPSEEK_API_KEY
- **说明**：DeepSeek API 密钥
- **必需**：是
- **获取方式**：https://platform.deepseek.com/

### VITE_DEEPSEEK_API_URL
- **说明**：API 端点地址
- **默认值**：https://api.deepseek.com/v1/chat/completions
- **必需**：否

### VITE_DEEPSEEK_MODEL
- **说明**：使用的模型名称
- **默认值**：deepseek-chat
- **必需**：否

### VITE_DEEPSEEK_TEMPERATURE
- **说明**：生成温度（0-1，越高越随机）
- **默认值**：0.7
- **必需**：否

### VITE_DEEPSEEK_MAX_TOKENS
- **说明**：最大生成长度
- **默认值**：4000
- **必需**：否

## ⚠️ 注意事项

### 1. 环境变量命名
- Vite 项目的环境变量必须以 `VITE_` 开头
- 否则不会被暴露给客户端代码

### 2. 重启服务器
- 修改 .env 文件后，**必须重启开发服务器**才能生效
- 热更新不会重新加载环境变量

### 3. 不要提交 .env
- .env 文件已添加到 .gitignore
- 确保不要手动添加到 Git
- 只提交 .env.example 作为示例

### 4. 备用机制
- 即使 .env 文件不存在，系统仍然可以工作
- 会使用硬编码的备用值
- 但建议使用环境变量
